54 int gopt (char **argv, struct option *options);
55 /*
56  The function gopt() takes an argument vector argv, which has the same
57  form as the second argument to main(). It removes from the vector all
58  options and option-arguments, leaving only the program name, the
59  operands and a null pointer at the end. It returns the updated
60  argument count argc. It does not need to know the initial value of
61  argc because the input vector must be null-terminated.
62 
63  It also takes an array of struct option, the elements of which are
64  used for specifying the options which are to be recognized and for
65  returning details of which options were present in the argument
66  vector.
67 
68  The members of struct option are used as follows:
69 
70  On input, short_name is the single character that follows "-" to make
71  a short option, or zero if there is no short form of this option.
72 
73  On input, long_name is the name of the long option excluding the "--",
74  or NULL if there is no long form of this option.
75 
76  On return short_name and long_name are unaltered.
77 
78  count may be uninitialized on input. On return it is set to the
79  number of times the option was specified. If the option is not
80  present, this is zero. If an option has both a short and long name,
81  this is the sum of counts for both forms.
82 
83  argument may be uninitialized on input. On return it is set to the
84  last option-argument specified to this option, or NULL if no
85  option-argument was specified with the last occurrence of this
86  option, or the option was never specified.
87 
88  flags is the bitwise-or of certain constants. On input flags must
89  contain exactly one of:
90  GOPT_ARGUMENT_FORBIDDEN if the option does not take an option-argument
91  GOPT_ARGUMENT_REQUIRED if it always requires an option-argument
92  GOPT_ARGUMENT_OPTIONAL if it may or may not take an option-argument
93 
94  If flags contains GOPT_ARGUMENT_REQUIRED, it may also contain
95  GOPT_ARGUMENT_NO_HYPHEN. If the option argument is not allowed to
96  start with a '-', but the next member of the argument vector does
97  start with one where an option argument was expected, then this flag
98  causes it to be interpreted as another option, leaving the argument
99  to the preceding option NULL. The application can then detect the
100  NULL and give an error message like "missing option argument" which
101  may be preferable to treating the next program argument as an option
102  argument and then giving a less helpful message when the option
103  argument is found to be invalid.
104 
105  flags may also optionally contain GOPT_REPEATABLE. This is ignored
106  by the option parsing code, but may be used when checking the value
107  returned in count, where if this flag is specified then the
108  application will presumably report an error if count is greater than
109  one.
110 
111  On return, the initial value of flags is still present, and
112  additionally GOPT_SEEN_SHORT_WITH and/or GOPT_SEEN_SHORT_WITHOUT are
113  set if the short name was used at least once with or without an
114  option-argument, respectively, and GOPT_SEEN_LONG_WITH and/or
115  GOPT_SEEN_LONG_WITHOUT are set if the long name was used.
116 
117  On input there must be zero or more elements in the options array
118  initialized as above. These must be followed by one element where
119  flags is exactly GOPT_LAST. All other members of this element may be
120  uninitialized on input. On return the count member of this element
121  will be set to the total number of unrecognized options present.
122  The short_name member will be set to the first unrecognized short
123  option, or zero if there were none. The long_name will be set to
124  point to the start of the name of the first unrecognized long option,
125  or NULL if there were none. For completeness the argument member
126  will be set to the option argument of the last unrecognized option
127  (the same as with other options) but this is probably useless to the
128  application.
129 */
