Network Logger.

Network logger using czmq. Use a different abl\+\_\+log for each thread and you should be ok. There is a possibility of a multi-\/thread logger at some point in the future, but not now. It uses a zmq publish socket under the hood, and you can use zmq subscribe to subscribe to the log.

When subscribing, you can subscribe to any of the levels; each level is a different topic. Or subscribe to all of them using a blank subscription of {\ttfamily \char`\"{}\char`\"{}}. Available topics are\+: \char`\"{}\+T\+R\+A\+C\+E\char`\"{} \char`\"{}\+D\+E\+B\+U\+G\char`\"{} \char`\"{}\+I\+N\+F\+O\char`\"{} \char`\"{}\+W\+A\+R\+N\char`\"{} \char`\"{}\+E\+R\+R\+O\+R\char`\"{} \char`\"{}\+F\+A\+T\+A\+L\char`\"{}

You can use printf()-\/like syntax with the error messages, see example.

For an example of subscribing to the log, see {\ttfamily \mbox{\hyperlink{ab__loggerclient_8h}{ab\+\_\+loggerclient.\+h}}}.

This is a single-\/file-\/library. You may include the file anywhere in your source. Only once in your code, define the following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#define AB\_LOGGER\_SRC}}
\DoxyCodeLine{\textcolor{preprocessor}{\#include "{}ab\_logger.h"{}}}
\end{DoxyCode}


This will include the source.
